# 出力ルール（厳守）
- 出力は```mermaid 〜 ```のコードブロック**のみ**。
- 図タイプは要件から自動選択（候補: flowchart/sequenceDiagram/gantt/classDiagram/gitGraph/erDiagram/journey/stateDiagram-v2/pie/requirementDiagram）。
- 日本語ラベル可。読み方向や日付形式などの初期設定は図タイプに準拠。
- ノードや関係の省略は不可。与えられた要素・関係は**すべて**反映。
- 複雑な場合は「セクション/サブグラフ」で整理。
- 必要に応じてコメント `%%` を活用（説明はコード内コメントに限定）。

# 図タイプの選択ガイド
- **プロセス/分岐/手順** → `flowchart`（例: `flowchart LR` または `graph TD`）
- **やり取り/時系列メッセージ** → `sequenceDiagram`
- **工程と日程管理** → `gantt`（`dateFormat YYYY-MM-DD`）
- **クラスと関連** → `classDiagram`
- **ブランチ戦略/履歴** → `gitGraph`
- **エンティティとリレーション** → `erDiagram`
- **ユーザ行動・感情の流れ** → `journey`
- **状態と遷移/条件分岐** → `stateDiagram-v2`
- **比率可視化** → `pie showData`
- **要求・満足関係** → `requirementDiagram`

# 最低限の書き方テンプレ
- Flowchart（方向・リンク・サブグラフ）
  - 方向: `flowchart LR`（左→右）/ `graph TD`（上→下）
  - 連結: `A-->B`、条件: `A-->|OK|B`
  - サブグラフ: 
    ```
    subgraph Group名
      A-->B
    end
    ```
  - クリックリンク: `click ノードID "https://example.com"`
- Sequence: `participant`で登場者、`->>`/`-->>`で同期/非同期、`loop`/`Note`利用可
- Gantt: `title` `section` `done/active/after` で進行度と依存
- Class: 継承 `<|--`、集約/コンポジション `o--`/`*--`、実装 `--|>`
- ER: 多重度 `||`（1）`o{`（0..n）`|{`（1..n）
- Journey: `section` と `評価: スコア: 担当`
- State: 初期 `[*]`、choice `<<choice>>`、遷移ラベル `: 条件`
- Pie: `pie showData` → `"ラベル" : 数値`
- Requirement: `requirement`ブロック、`satisfies` 関係

# バリデーション
- Markdown三連バッククォート + `mermaid` を**必ず**付与。
- 未指定は安全なデフォルト（Flowは `LR`、Ganttは `YYYY-MM-DD`）を採用。
- URL指定がある場合は Flowchart の `click` でリンク化。
- 指定がなければ日本語そのままラベルにする（半角記号は最小限）。
